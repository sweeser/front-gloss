1. Как задать имя пользователя и адрес электронной почты

Имя пользователя необходимо для привязки коммитов к вашему имени. Это не то же самое, что имя пользователя вашей учётной записи GitHub, которое используется для входа в профиль на GitHub. Задать или изменить имя пользователя можно с помощью команды git config. Новое имя будет автоматически отображаться в последующих коммитах, отправленных на GitHub через командную строку. Если хотите скрыть своё реальное имя, можно использовать произвольный набор символов в качестве имени пользователя Git.

Пример кода:

git config --global user.name "John Smith"

Также командой git config можно изменять адрес электронной почты, привязанный к вашим коммитам Git. Новый адрес электронной почты будет автоматически отображаться во всех дальнейших коммитах, отправленных на GitHub через командную строку.

Пример кода:

git config --global user.email "jSmithexamlgmail.com"

2. Кэширование учётных данных

Для кэширования учётных данных можно использовать команду git config с флагом --global. Это позволит вам избежать необходимости вручного ввода имени пользователя и пароля при создании нового коммита.

Пример кода:

git config --global credential.helper cache

3. Инициализация репозитория

С помощью команды git init можно создать пустой репозиторий Git или вновь инициализировать уже существующий. При инициализации будет создана скрытая папка, в которой будут храниться все объекты и ссылки, используемые и создаваемые Git для ведения истории работы над проектом.

Пример кода:

git init

4. Добавление отдельных файлов или всех файлов в область подготовленных файлов

Чтобы добавить отдельный файл в область подготовленных файлов, используйте команду git add, указав имя файла. Например, замените somefile.js на имя вашего файла:

Пример кода:

git add somefile.js

Чтобы добавить все файлы и папки в область подготовленных файлов, используйте символ подстановки точки вместо имени файла:

Пример кода:

git add .

5. Проверка статуса репозитория

Чтобы просмотреть статус репозитория, используйте команду git status. Эта команда покажет информацию о подготовленных, неподготовленных и неотслеживаемых файлах.

Пример кода:

git status

6. Внесение изменений однострочным сообщением или через редактор

При создании коммита в репозитории можно добавить однострочное сообщение с помощью команды git commit и флага -m. Сообщение вводится в кавычках непосредственно после флага.

Пример кода:

git commit -m "Your text for commit"

Также можно открыть текстовый редактор в терминале для написания полного сообщения коммита, которое может состоять из нескольких строк и подробно описывать внесённые изменения.

Пример кода:

git commit

7. Просмотр истории коммитов с изменениями

Чтобы просмотреть изменения, внесённые в репозиторий, используйте команду git log. Она отображает список последних коммитов в порядке их выполнения. Для подробного изучения изменений в каждом файле добавьте флаг -p.

Пример кода:

git log -p

8. Просмотр заданного коммита

Чтобы просмотреть полный список изменений, внесённых конкретным коммитом, используйте команду git show с указанием идентификатора или хеша коммита. Хеш уникален для каждого коммита в вашем репозитории.

Пример кода:

git show 1af17e73721dbe0c40011b82ed4bb1a7dbe3ce29

Вы также можете использовать сокращённый хеш.

Пример кода:

git show 1af17e

9. Просмотр изменений до коммита

Для просмотра списка изменений, внесённых в репозиторий, используйте команду git diff. По умолчанию отображаются только изменения, не подготовленные для коммита.

Пример кода:

git diff

Чтобы просмотреть подготовленные изменения, добавьте флаг --staged.

Пример кода:

git diff --staged

Также можно указать имя файла, чтобы просмотреть изменения, внесённые только в этот файл.

Пример кода:

git diff somefile.js

10. Удаление отслеживаемых файлов из текущего рабочего дерева

Для удаления файлов из текущего рабочего дерева и индекса используйте команду git rm. Например, чтобы удалить конкретный файл.

Пример кода:

git rm dirname/somefile.js

Вы также можете использовать маски файлов для удаления всех файлов, соответствующих определённому критерию.

Пример кода:

git rm dirname/*.html
